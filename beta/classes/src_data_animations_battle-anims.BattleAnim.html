    play

    • play(onSubstitute?: boolean, callback?: () => void): void

      Plays the animation between the defined user and target.

      Note: This assumes the user and target are defined, and will likely crash the game otherwise unless playRegardlessOfIssues is enabled. If playRegardlessOfIssues is enabled, animation frames targeting an undefined user or target are not played.

      Parameters

      • OptionalonSubstitute: boolean

        If true, and the target Pokemon has an active substitute, the animation will treat the substitute's sprite

      • Optionalcallback: () => void

        If defined, is executed immediately after the animation completes.

      Returns void

    playWithoutTargets

    • playWithoutTargets(
          targetInitialX: number,
          targetInitialY: number,
          frameTimeMult: number,
          frameTimedEventPriority?: 0 | 1 | 3 | 5,
          callback?: () => void,
      ): void

      Plays this animation ignoring frame data for "user" or "target" Pokemon. This only plays the graphic components of the animation.

      Parameters

      • targetInitialX: number

        The x-coordinate of the animation's start point, relative to userFocusX

      • targetInitialY: number

        The y-coordinate of the animation's start point, relative to userFocusY

      • frameTimeMult: number

        A multiplier for the delay between graphic frames

      • OptionalframeTimedEventPriority: 0 | 1 | 3 | 5

        The depth (or z-coordinate) of animated graphics:

        • 0 is behind all other sprites (except BG)
        • 1 is on top of player field
        • 3 is on top of both fields
        • 5 is on top of player sprite
      • Optionalcallback: () => void

        A function invoked immediately after the animation completes

      Returns void
